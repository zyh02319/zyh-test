## 项目概览

这是一个基于 `Vite + Vue 3` 的静态展示型前端项目，项目名是 `codex-shop`。

当前成品是一页式餐厅品牌站，品牌名称为 `Maple Table`。页面主题偏“城市餐叙 / 季节餐桌”风格，内容围绕餐厅介绍、招牌菜、到店信息和顾客评价展开。

项目体量很小，没有路由、状态管理、接口请求、后端代码，也没有拆分成多个 Vue 组件；目前主要是一个单页页面。

## 技术栈

- `Vue 3`
- `Vite`
- `@vitejs/plugin-vue`
- 原生 CSS

## 运行方式

`package.json` 中提供了 3 个脚本：

- `npm run dev`：启动本地开发环境
- `npm run build`：构建生产包
- `npm run preview`：预览构建结果

## 目录结构

项目当前核心文件很少，结构非常直接：

- `index.html`：HTML 入口，挂载 `#app`
- `src/main.js`：Vue 应用入口，挂载 `App.vue`，并引入全局样式
- `src/App.vue`：主页面内容，整个站点的主体模板都在这里
- `styles.css`：全局样式文件，负责页面布局、配色、排版和响应式
- `vite.config.js`：Vite 配置，当前只启用了 Vue 插件
- `dist/`：构建产物目录

## 页面内容结构

`src/App.vue` 当前是一个完整的一页式落地页，主要区块如下：

1. 顶部导航 `site-header`
2. Hero 首屏，含品牌文案、主视觉图片、按钮和订位信息
3. 三列亮点介绍 `highlights`
4. 本周招牌菜单 `menu-section`
5. 品牌故事 `story-section`
6. 到店信息 `visit-section`
7. 顾客评价 `reviews-section`
8. 页脚 `site-footer`

页面内容是纯静态数据，暂未接入后台或可配置数据源。

## 样式与视觉特征

`styles.css` 显示出当前项目的设计方向比较明确：

- 使用 CSS 变量统一管理颜色
- 暖色、米色、棕色系为主，强调餐厅质感
- 标题字体使用 `Noto Serif SC`
- 正文字体使用 `Inter`
- 大量使用 Grid 布局
- 桌面端和移动端均有响应式处理
- 卡片圆角统一较克制，视觉偏精致、安静、轻复古

## 外部依赖与资源

页面依赖了一些外部资源：

- Google Fonts
- Unsplash 图片资源

这意味着页面展示效果依赖外网资源是否可访问。

## 当前代码现状

我在通读时观察到这些事实：

- 项目可以看作“单组件静态官网”
- 所有主要页面内容都堆叠在 `src/App.vue`
- 样式集中在根目录的 `styles.css`，不是放在 `src/` 下
- 没有测试文件
- 没有 ESLint、Prettier、TypeScript 等工程化配置
- 没有资源本地化目录，例如 `public/`、`assets/` 图片资源目录

## 当前已发现的问题

最明显的问题是：项目中的中文文本存在乱码。

具体表现：

- `index.html` 中的标题和描述出现乱码
- `src/App.vue` 中大量中文文案出现乱码
- 当前 `AGENTS.md` 原始内容在读取时也呈现乱码

这通常意味着文件编码、编辑器保存编码，或者终端读取编码之间存在不一致。后续如果要继续维护这个项目，建议优先处理编码问题，否则文案修改和页面上线都会受影响。

## 后续维护建议

如果后面继续开发，这个项目比较适合按下面方向演进：

- 先修复中文编码问题，统一为 `UTF-8`
- 将 `App.vue` 拆分为多个语义化组件
- 把菜单、营业时间、评价等内容抽成数据对象
- 将图片资源改为本地可控资源或稳定 CDN
- 补充基础工程化能力，例如格式化、Lint

## 给后续代理/协作者的提示

- 这是一个小型 Vue 静态站，不要默认它有复杂业务逻辑
- 修改页面内容时，优先查看 `src/App.vue`
- 修改视觉时，优先查看 `styles.css`
- 如果发现中文异常，先检查文件编码，不要急着按“文本内容错误”处理
